Core Highlights

Enterprise-Grade Features

Dynamic RBAC Roles

Enforces strict Role-Based Access Controls. The frontend automatically filters sidebar menus and locks/unlocks REST routes based on Administrator, Sales Manager, and Employee credentials.

Dual Custom Control Panels

Dynamic workspace switching. Administrators receive a charcoal-slate System Control Hub, while Sales Managers get a friendly teal Sales Dashboard, preventing visual overload.

Stock Telemetry Service

Automated inventory stock deductions upon sales order confirmation, tracking warning limits. Emits real-time warning highlights on the workspace to prevent stockout operations.

PDF Document Exports

Writers module compiling clean PDF printouts of Quotations and Billing Invoices using ReportLab, containing auto-calculated discounts, VAT columns, and balances.

Excel Analytics Reports

Analytical downloads compile ledger data directly into corporate Excel sheets (.xlsx), capturing detailed sales history volumes and inventory valuations.

Fault-Tolerant DB Fallback

Startup check utility. On startup, the backend tests the PostgreSQL connection. If unreachable, it seamlessly falls back to a local SQLite instance, ensuring absolute uptime.

Getting Started

Run the Application Locally

You can boot the full server and database using either native Python or containerized Docker-Compose configurations. Follow the code steps on the right.

Requires Python 3.11.x installed. The application will boot up on a local SQLite fallback database (`sales_order_system.db`) if no PostgreSQL configurations are declared inside the `.env` file.

Requires Docker Desktop. Executing `docker-compose up` builds the Python Flask app container and boots up a local PostgreSQL 15 database instance inside a virtual network automatically.
PowerShell / Bash Console
# 1. Clone the repository and navigate inside
git clone https://github.com/your-username/sales-order-management.git
cd sales-order-management

# 2. Install backend dependencies
pip install -r requirements.txt

# 3. Start the production WSGI server
python app.py
